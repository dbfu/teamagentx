# 模型配置

模型配置决定助手调用哪个 AI 服务。只要模型配置不可用，助手就可能无法回答、无法生成图片、无法语音播报，或者系统助手无法帮你自动创建配置。

![模型管理](/images/user-guide/model-management.png)

## 什么时候需要配置模型

以下情况都需要先进入“模型”页：

- 第一次使用 TeamAgentX，还没有默认文本模型。
- 新建助手时没有可选模型供应商。
- 想把助手切换到 DeepSeek、OpenAI 兼容、Claude 兼容或其他服务。
- 想启用图片生成能力，需要添加图片模型。
- 想使用语音播报或语音识别，需要添加语音模型。
- 想把一台机器上的模型配置迁移到另一台机器。

## 模型页能做什么

| 功能 | 说明 |
| --- | --- |
| 搜索模型 | 按模型名称、模型 ID、API 地址搜索已有配置 |
| 类型筛选 | 查看全部、文本、图片、视频、语音模型 |
| 新增模型 | 手动填写模型名称、API URL、API Key、模型 ID 等字段 |
| AI 创建 | 把一段服务商配置描述交给 AI 自动解析，生成表单草稿 |
| 导入 JSON | 批量导入之前导出的模型配置 |
| 导出 JSON | 导出当前模型配置，便于迁移或备份 |
| 测试连接 | 验证 API URL、API Key、协议和模型名是否可用 |
| 复制配置 | 以已有模型为模板创建一个副本 |
| 设为默认 | 指定系统默认文本模型，或默认 STT 语音识别模型 |
| 启用/停用 | 临时禁用某个模型，不删除配置 |
| 删除模型 | 删除配置；关联助手会失去对应模型能力 |

## 新增文本模型

点击“新增模型”会打开模型表单。

![新增模型](/images/user-guide/model-create-dialog.png)

文本模型最常用字段如下：

| 字段 | 必填 | 怎么填 |
| --- | --- | --- |
| 名称 | 是 | 只给自己看的名称，例如“DeepSeek Chat”“Claude Sonnet” |
| 模型类型 | 是 | 普通对话和代码助手选“文本” |
| API URL | 是 | 服务商给出的接口地址，例如 OpenAI 兼容服务通常是 `/v1` 结尾 |
| API Key | 是 | 服务商密钥；编辑时如显示遮罩且不想改密钥，可以留用原密钥 |
| 模型 | 是 | 服务商文档里的模型 ID，例如 `deepseek-chat`、`gpt-4.1` |
| API 协议 | 是 | Claude/Anthropic 兼容选 Anthropic；OpenAI 兼容选 OpenAI |
| 设为默认模型 | 否 | 开启后，新助手和系统助手会优先使用它 |

### 协议怎么选

| 协议 | 适合场景 |
| --- | --- |
| Anthropic | Claude 官方或兼容 Anthropic Messages API 的服务。适合需要 thinking、prompt caching 等 Claude 特性的场景。 |
| OpenAI | OpenAI、DeepSeek、OpenRouter、火山方舟、通义百炼等 OpenAI 兼容接口。适合大多数第三方模型。 |

如果不确定，先看服务商文档写的是不是“OpenAI compatible”。是的话通常选 OpenAI。

## 新增图片模型

图片模型用于助手生成图片。创建时把“模型类型”切到“图片”。

图片模型比文本模型多两个关键字段：

| 字段 | 说明 |
| --- | --- |
| 图片供应商 | 可选 OpenAI、APIMart、OpenRouter、Gemini、智谱、百炼、xAI、火山等 |
| 调用方式 | 同步返回图片、异步任务轮询，或自动选择 |

注意事项：

- OpenRouter 需要填写支持图片输出的模型，不能填普通文本模型。
- 智谱图片模型建议填写实际图片模型 ID，例如 `glm-image`。
- xAI 推荐使用官方当前图片模型 ID，例如 `grok-imagine-image-quality`。
- 助手要用图片能力，还需要在助手配置里打开“图片生成能力”并选择图片模型。

## 新增语音模型

语音模型用于 TTS 朗读和 STT 语音识别。创建时把“模型类型”切到“语音”。

| 字段 | 说明 |
| --- | --- |
| 语音用途 | 可选“仅 TTS 朗读”“仅 STT 识别”“TTS + STT” |
| TTS 模型 | 用于朗读助手消息 |
| STT 模型 | 用于识别语音输入；留空时可能复用 TTS 模型 |
| 默认 STT | 语音模型设为默认时，含义是默认语音识别模型 |

如果只是想让助手消息自动播报，需要 TTS；如果想把用户语音转文字，需要 STT。

## AI 创建模型配置

“AI 创建”适合你有一段 API 信息，但不确定怎么填表单。

你可以输入类似：

```text
我有一个 DeepSeek API，地址是 https://api.deepseek.com，API Key 是 sk-xxx，模型是 deepseek-chat。
```

系统会尝试解析名称、API URL、API Key、模型和协议，然后打开表单让你确认。AI 创建不是直接保存，仍然需要你检查后点击“创建”。

使用限制：

- 需要已有可用默认文本模型，否则 AI 没有能力帮你解析。
- API Key 属于敏感信息，只建议在本地可信环境使用。
- AI 解析后一定要检查协议和模型名，避免服务商格式被误判。

## 导入和导出 JSON

导出 JSON 会生成类似 `teamagentx-model-providers-YYYY-MM-DD.json` 的文件，包含当前模型配置。

导入时系统会检查：

- 文件是否是合法 JSON。
- 是否包含模型列表。
- 每项是否有名称、API URL、API Key、模型。
- 图片模型是否包含图片供应商和调用方式。

导入策略：

- 非默认模型会先导入。
- 默认模型会后导入，避免默认状态被前面的配置覆盖。
- 部分失败时会显示成功数量和失败原因。

安全提醒：导出的 JSON 包含 API Key，不要提交到 Git 仓库，不要发给不可信的人。

## 连接测试和排查

配置后建议立即点“测试连接”。常见失败原因如下：

| 现象 | 优先检查 |
| --- | --- |
| 401/403 | API Key 是否正确、是否有余额或权限 |
| 404 | API URL 是否多写/少写 `/v1`，模型名是否拼错 |
| 超时 | 网络代理、防火墙、服务商状态 |
| 协议错误 | Claude 服务是否误选 OpenAI，OpenAI 兼容服务是否误选 Anthropic |
| 助手仍不可用 | 模型是否已启用，助手是否绑定了该模型 |

## 建议配置方案

新手可以先这样配置：

| 用途 | 最小配置 |
| --- | --- |
| 普通聊天和代码任务 | 一个文本模型，设为默认 |
| 图片生成 | 一个图片模型，并在需要的助手里打开图片能力 |
| 语音播报 | 一个语音 TTS 模型，然后到助手详情的“语音”页启用 |
| 多服务备份 | 同类型模型配置 2 个，一个主用，一个停用备用 |

不要一开始配置太多模型。先保证一个文本模型稳定可用，再逐步添加图片和语音能力。
