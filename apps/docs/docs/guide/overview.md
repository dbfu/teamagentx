# 总览与上手路径

TeamAgentX 可以理解为一个“AI 同事群聊”。你把不同角色的助手拉进同一个群，给它们分配模型、技能、工作目录和群规则，然后在聊天里通过 `@助手名` 发任务。

![群聊工作区](/images/user-guide/chat-room-clean.png)

## 你需要先理解的 5 个概念

| 概念 | 通俗理解 | 你在界面里会做什么 |
| --- | --- | --- |
| 模型 | AI 的大脑，决定助手用哪个服务回答和执行任务 | 在“模型”页添加 API URL、API Key、模型名，并测试连接 |
| 技能 | 助手的能力包，类似“工具说明书”或“专项工作方法” | 在“技能”页导入技能，再安装给指定助手 |
| 助手 | 有名称、角色、提示词、模型和技能的 AI 成员 | 在“助手”页创建、分类、编辑、停用、复制或发起快速对话 |
| 群聊 | 一个协作空间，包含消息历史、工作目录、群规则和多个助手 | 在“消息”页创建群聊、添加助手、设置触发模式 |
| 工作目录 | 助手运行命令和读写文件的本地目录 | 在群设置或快速对话中指定，避免助手操作错项目 |

## 第一次使用的推荐路径

1. 进入应用后先完成初始化向导，或者注册/登录本地账号。
2. 打开“模型”，至少配置一个可用文本模型，并设置为默认模型。
3. 打开“助手”，创建一个明确角色的助手，例如“工程师”“文档助手”“测试助手”。
4. 打开“技能”，按需要给助手安装技能，不要一开始把所有技能都装给所有助手。
5. 回到“消息”，创建一个项目群聊，选择项目工作目录并添加助手。
6. 在输入框里发送 `@助手名 任务内容`，观察执行过程、任务队列和执行详情。

## 主界面导航

左侧导航是 TeamAgentX 的主要入口。

| 入口 | 适合做什么 |
| --- | --- |
| 消息 | 创建群聊、进入项目群、发消息、管理群成员和任务 |
| 助手 | 创建助手、编辑提示词、安装技能、发起快速对话 |
| 技能 | 查看共享技能、导入外部技能、安装技能到助手 |
| 模型 | 添加文本/图片/语音模型，测试连接，导入导出配置 |
| 频道 | 管理外部平台机器人，把飞书、Telegram 等消息接入群聊 |
| 设置 | 修改个人资料、主题、提示音、移动端连接和本地工具设置 |

## 推荐的最小可用配置

| 项目 | 推荐配置 |
| --- | --- |
| 模型 | 先配置一个稳定的文本模型，并设为默认 |
| 助手 | 先建 2 到 3 个：总控、工程师、质检或文档助手 |
| 技能 | 只给当前角色需要的技能，减少无关能力干扰 |
| 群聊 | 一个项目一个群，工作目录指向项目根目录 |
| 触发模式 | 新手推荐“协调模式”；想完全手动控制时用“手动模式” |
| 群规则 | 写清楚语言、范围、验证要求和禁止事项 |

## 常用任务写法

清楚的任务通常包含目标、范围、约束和交付物。

```text
@工程师 请阅读当前项目的 README 和 package.json，总结本地启动步骤。先不要修改文件。
```

```text
@文档助手 请把这次群聊中的结论整理成面向普通用户的说明，输出 Markdown。
```

```text
@测试助手 请检查最近一次代码改动的风险点，重点看是否缺少验证步骤。
```

## 使用建议

- 重要任务尽量显式 `@助手名`，减少系统猜测。
- 修改代码前确认工作目录正确，尤其是桌面端同时打开多个项目时。
- 助手答非所问时，先检查群规则、默认助手、触发模式和上下文。
- 多助手协作时，让一个“总控”先拆任务，再分别 `@` 专项助手。
- 如果只是临时问一个助手问题，用“快速对话”比新建群更轻量。
