# 技能配置

技能是助手的能力包。它通常包含 `SKILL.md` 和相关脚本/模板，告诉助手“遇到某类任务时应该怎么做”。技能不会自动让所有助手变强，只有安装到某个助手后，该助手才会在执行时看到它。

![技能管理](/images/user-guide/skill-management.png)

## 技能页能做什么

| 功能 | 说明 |
| --- | --- |
| 查看共享技能 | 看到已经导入到 TeamAgentX 共享技能库的技能 |
| 搜索技能 | 按名称、描述快速过滤技能 |
| 查看技能详情 | 打开技能文件树，查看 `SKILL.md` 和相关文件 |
| 安装到助手 | 把某个技能安装给一个或多个助手 |
| 导入外部技能 | 扫描 Claude Code、Codex、OpenClaw、Agents 等本地技能目录 |
| 软连接导入 | 共享目录指向外部技能，外部更新会同步 |
| 复制导入 | 把技能完整复制到本地共享目录，之后独立管理 |
| 打开目录 | 在桌面客户端里打开技能目录，Web 版可能不支持 |

## 技能目录在哪里

共享技能目录显示为：

```text
~/.teamagentx/skills
```

这是 TeamAgentX 自己管理的技能库。外部 AI 工具的技能不会自动变成共享技能，需要你手动导入。

## 技能和助手是什么关系

可以把助手理解成“人”，技能理解成“工作手册”。

| 配置 | 作用 |
| --- | --- |
| 助手提示词 | 决定助手是谁、性格是什么、平时怎么工作 |
| 技能 | 决定遇到某类任务时，助手应该按哪套流程做 |
| 模型 | 决定助手使用哪个 AI 服务思考和输出 |
| 群规则 | 决定当前群里所有助手共同遵守什么规则 |

一个技能导入到技能库后，并不会自动给所有助手使用。你需要把技能安装到具体助手上。比如“代码审查技能”适合安装给测试助手或代码审查助手，不一定适合安装给文案助手。

## 导入外部技能

点击“导入外部技能”后，系统会扫描常见外部技能目录。

![导入外部技能](/images/user-guide/skill-import-dialog.png)

当前支持的来源包括：

| 来源 | 常见目录 |
| --- | --- |
| Claude Code | `~/.claude/skills` |
| Codex | `~/.codex/skills` |
| OpenClaw | `~/.openclaw/skills` |
| Agents | `~/.agents/skills` |

导入方式有两种：

| 方式 | 优点 | 适合场景 |
| --- | --- | --- |
| 软连接导入 | 不复制文件，占用空间小，外部技能更新后同步生效 | 你希望继续维护外部技能，并让 TeamAgentX 使用同一份 |
| 复制导入 | 独立保存，不受外部目录删除或修改影响 | 你希望固定当前版本，避免外部更新改变助手行为 |

Windows 上符号链接能力可能受权限限制，界面会优先使用复制导入。

### 软连接和复制怎么选

如果你不知道选哪个，建议选“复制导入”。复制导入更稳定，外部工具删掉原目录后，TeamAgentX 里仍然有这份技能。

| 你的情况 | 推荐 |
| --- | --- |
| 只是普通使用，不想被外部变化影响 | 复制导入 |
| 你自己维护这份技能，希望外部更新后 TeamAgentX 也同步 | 软连接导入 |
| 公司团队发了一份固定技能包 | 复制导入 |
| 技能还在频繁修改调试 | 软连接导入 |

软连接导入不是“复制一份”，而是“指向外部原文件”。外部文件改了，TeamAgentX 里看到的也会变。

## 安装技能到助手

在技能卡片上点击“安装”，会打开选择助手弹框。你可以：

- 勾选一个或多个助手。
- 取消勾选已经安装的助手，表示从该助手卸载。
- 保存后系统会更新技能和助手之间的安装关系。

安装成功后，技能卡片会显示“已安装到”的助手名称。助手详情页的“Skills”页也会显示该助手已有技能。

### 安装前怎么判断适不适合

安装技能前建议看三点：

1. 技能描述是否和助手职责匹配。
2. `SKILL.md` 里有没有会运行命令、读写文件或访问外部服务的要求。
3. 技能是否会和助手已有技能冲突。

例如：

| 助手 | 适合安装 | 不建议安装 |
| --- | --- | --- |
| 工程师助手 | 前端开发、后端开发、调试、验证 | 小说写作、社群运营 |
| 文档助手 | 文档写作、截图整理、发布说明 | 数据库迁移、服务器运维 |
| 总控助手 | 任务拆解、项目管理、需求澄清 | 太多具体技术实现技能 |
| 测试助手 | 代码审查、测试设计、风险检查 | 图片生成、营销文案 |

## 在助手页安装技能

除了在“技能”页安装，也可以在“助手”页或助手详情里操作：

1. 进入“助手”页。
2. 找到目标助手。
3. 打开卡片菜单，点击“安装 Skills”。
4. 选择技能库里的技能，批量安装。

助手安装弹框还提供“对话安装”。你可以描述想要的能力，系统会跳转到技能安装助手，由它帮助查找和安装相关技能。

## 查看技能内容

点击技能卡片可以查看详情。详情窗口左侧是文件树，右侧是文件内容。

建议重点看：

- `SKILL.md`：技能触发条件、工作流程、注意事项。
- `scripts/`：技能是否会运行脚本。
- `assets/` 或模板文件：技能是否依赖固定模板。
- 技能描述：是否和你的助手角色匹配。

不要盲目安装来源不明的技能。技能可能指导助手运行命令或读写文件，安装前应确认它的内容可信。

## 技能详情怎么看

普通用户不需要完全看懂技能里的所有脚本，但至少应该会看 `SKILL.md`。

重点关注这些内容：

| 内容 | 为什么重要 |
| --- | --- |
| 触发条件 | 决定助手什么时候会使用这个技能 |
| 工作流程 | 决定助手会按哪些步骤完成任务 |
| 禁止事项 | 决定助手哪些事情不能做 |
| 需要的工具 | 可能涉及命令行、浏览器、文件读写或外部服务 |
| 输出要求 | 决定最终回复是报告、文件、表格还是代码 |

如果一个技能写着会删除文件、提交代码、发布内容或调用外部平台，安装前要更谨慎，最好只装给明确需要这些能力的助手。

## 技能数量怎么控制

技能越多不一定越好。一个助手拥有太多技能，会出现这些问题：

- 助手不知道该优先使用哪个流程。
- 上下文变长，响应更慢。
- 与当前任务无关的规则干扰判断。
- 不同技能的要求互相冲突。

推荐做法：

| 助手类型 | 推荐技能 |
| --- | --- |
| 工程师 | 编码、调试、验证、技术栈相关技能 |
| 测试/质检 | code review、验证、风险检查技能 |
| 文档助手 | 写作、文档、截图整理、结构化输出技能 |
| 总控/协调 | 任务拆解、需求澄清、项目管理技能 |

## 创建新技能

普通用户通常不需要手写技能文件。推荐在群聊里 `@技能管理`，让系统助手协助创建。

示例：

```text
@技能管理 帮我创建一个“发布检查”技能，用于在上线前检查构建、测试、环境变量和回滚说明。
```

创建后可以回到“技能”页查看、安装到助手。

## 一个新技能从创建到使用

完整流程如下：

1. 在群聊里让技能管理助手创建技能。
2. 回到“技能”页刷新列表。
3. 打开技能详情，检查 `SKILL.md` 内容是否符合预期。
4. 点击“安装”，选择一个或多个目标助手。
5. 回到群聊，用符合触发条件的任务测试。
6. 如果助手没有使用技能，调整任务描述或技能触发条件。

测试技能时，任务要写得明确一些。例如不要只写“帮我看看”，而是写：

```text
@测试助手 请使用代码审查技能检查最近一次改动，输出风险点、影响范围和建议验证命令。
```

这样助手更容易判断应该启用对应技能。

## 常见问题

### 导入后找不到技能

先点“刷新”，再检查外部目录里是否真的有 `SKILL.md`。没有 `SKILL.md` 的目录不会被当作有效技能。

### 安装后助手没有使用技能

检查是否安装到了正确助手，任务描述是否符合技能触发条件。很多技能只有在特定任务类型下才会被助手主动使用。

### 软连接导入后内容变了

这是软连接导入的正常行为。外部技能目录更新后，TeamAgentX 看到的也是新内容。如果希望固定版本，请使用复制导入。

### Web 版打不开目录

打开本地目录属于桌面能力。纯浏览器模式可能只能复制路径，不能直接打开 Finder/资源管理器。
