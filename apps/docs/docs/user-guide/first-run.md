# 首次设置

首次打开 TeamAgentX 时会进入初始化向导。这个流程的目标是检测本地 AI 工具、准备默认模型，并创建第一个用户账号。

## 设置流程

1. 在欢迎页点击 **开始设置**。
2. 进入 **检测本地 AI 工具**，系统会检查 Claude、Codex 等工具状态。
3. 在 **已安装的工具** 中选择默认引擎。
4. 如果工具未安装，可以在同一页面点击 **安装**，安装完成后点击 **重新检测**。
5. 进入 **配置默认模型**，选择配置方式。
6. 创建账号，设置用户名、密码和头像。
7. 完成后进入主界面。

## 本地 AI 工具

工具检测页会把工具分为已安装和未安装两类。

- 已安装工具可以直接选为默认引擎。
- 未安装工具会显示安装按钮和安装日志。
- 如果工具状态不准确，点击 **重新检测**。

默认引擎会影响初始化时创建的默认 Agent 配置。后续仍可以在助手创建或编辑时选择其他 Agent。

## 模型配置方式

### 使用本地配置

如果系统检测到本地 CLI 配置，可以选择 **使用本地配置**。

- Claude 会使用本地 Claude 配置，可以选择本地模型或使用默认模型。
- Codex 会使用本地 `auth.json` 中的 API Key，可以选择 Codex 模型或使用本地默认模型。
- 模型留空时走本地工具自己的默认模型。

### 手动配置

选择 **手动配置** 时，需要填写：

- **API URL**：模型服务地址。
- **API Key**：访问密钥，必填。
- **模型名称**：模型 ID，必填。
- **API 协议**：由所选引擎自动确定。Codex 使用 OpenAI 协议，Claude 使用 Anthropic 协议。

手动配置会创建默认模型配置，后续可以在 **模型** 页面继续编辑。

## 账号规则

创建账号时：

- 用户名长度需要在 2 到 20 个字符之间。
- 密码长度至少 4 个字符。
- 两次密码必须一致。

登录后，个人信息可以在 **设置** 页面修改。

## 完成后的检查

首次设置完成后建议检查：

1. 进入 **模型** 页面，确认至少有一个启用的文本模型。
2. 进入 **助手** 页面，确认默认助手存在并处于启用状态。
3. 创建一个群组，发送 `@助手名 你好` 测试回复。

如果助手没有响应，优先检查模型配置、Agent 工具状态和群聊触发模式。
