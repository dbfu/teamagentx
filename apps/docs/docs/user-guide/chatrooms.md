# 群聊与消息

群聊是 TeamAgentX 的核心工作空间。用户在群聊中发送消息，通过 `@助手名` 指定 AI 助手执行任务，也可以让多个助手协作。

## 创建群组

点击左侧 **+**，选择 **创建群组**。

可配置项：

- **群组名称**：必填。
- **头像**：从内置头像中选择。
- **群组描述**：用于说明群聊用途。
- **工作目录**：群内所有助手共享的运行目录。留空时使用默认群目录。
- **选择助手**：选择要加入群聊的助手。
- **注入群历史消息**：每个选中助手默认开启。开启后助手会获取群聊上下文。

创建完成后会自动进入消息页。

## 发送消息和 @ 助手

在输入框中输入 `@` 会出现助手候选列表，选择后插入提及。

示例：

```text
@代码助手 请检查当前项目的测试失败原因。
```

可以同时提及多个助手：

```text
@产品助手 @代码助手 请评估这个需求是否容易实现。
```

如果消息中提及了过多可调度助手，系统会提示你减少提及数量，避免一次性触发过多执行。

## 触发模式

群设置中可以选择 **助手触发模式**。

### 协调模式

创建群组时默认使用协调模式。

- 用户 `@` 指定助手时，直接触发被提及助手。
- 没有 `@` 的用户消息，以及助手消息，会交给系统内置协调助手派发。
- 适合多人多助手协作场景。

### 自由协作

- 有 `@` 时触发指定助手。
- 没有 `@` 的用户消息由 **默认接收助手** 处理。
- 适合固定一个主助手，偶尔点名其他助手的群聊。

### 手动模式

- 用户 `@` 指定助手时触发。
- 助手消息里的 `@` 只作为提及，不会继续自动派发。
- 适合希望严格控制每次执行的群聊。

## 群设置

点击群标题、头像或右上角设置按钮打开 **群设置**。

可管理：

- **群组名称**
- **头像**
- **群组描述**
- **工作目录**
- **默认接收助手**
- **助手触发模式**
- **群规则**
- **外部平台机器人绑定**
- **清空消息**
- **删除对话**

### 工作目录

工作目录决定助手执行文件操作和命令时的基础位置。

- 设置了群工作目录后，群内助手共享该目录。
- 桌面端可以选择目录、打开目录、复制路径，或用指定终端打开。
- 留空时使用默认目录：`~/.teamagentx/workspace/<群聊ID>`。

### 群规则

群规则会注入到群内所有助手上下文中。适合写：

```text
- 所有回复使用中文。
- 修改代码前先说明计划。
- 不确定的信息需要明确说明。
- 输出方案时同时给出风险。
```

## 成员管理

点击群标题旁的成员数量或右上角 **添加助手** 进入成员管理。

成员会分组显示：

- **群主**
- **普通助手**
- **系统助手**

常用操作：

- 点击助手查看详情。
- 点击 `@` 按钮把助手插入输入框。
- 右键助手也可以插入提及。
- 移除非群主、非系统助手。
- 添加当前不在群内的活跃助手。

快速对话群聊不允许添加新助手。

## 任务执行过程

助手执行时，右侧面板可以查看：

- **流式输出**：实时查看思考、工具调用和输出。
- **任务队列**：查看某个助手排队中的任务。
- **任务看板**：查看群内所有助手当前任务状态。
- **执行详情**：查看历史执行事件、耗时、Token 和工具结果。
- **查看上下文**：查看助手最近上下文、系统提示词和工具调用信息。
- **历史执行结果**：查看某个助手在当前群聊里的历史记录。

如果任务正在执行，顶部会显示 **停止所有任务** 按钮；也可以在流式面板中停止单个助手。

## 更多功能

群聊右上角 **更多** 菜单包含：

- **群规则**
- **截图聊天记录**
- **定时任务**
- **历史记录**
- **清空消息**

如果群工作目录中存在 `package.json`，顶部还会显示脚本菜单。桌面端可以把脚本命令发送到终端执行。

## 消息操作

消息支持：

- 回复指定消息。
- 点击提及跳转到助手详情。
- 查看执行详情。
- 截图聊天记录。
- 查看归档历史。
- 删除或批量选择消息。

收到未读消息时，左侧消息列表会显示未读数量；进入群聊后会自动标记已读。
