# 助手管理

助手是 TeamAgentX 中执行任务的 AI 成员。不同助手可以使用不同 Agent 工具、模型、提示词、技能和能力配置。

## 页面入口

左侧点击 **助手** 查看助手列表。也可以点击左侧 **+**，选择 **创建助手**。

助手详情页包含：

- **配置信息**
- **语音**
- **Skills**
- **会话记录**

系统内置助手可能不允许查看详情或修改配置。

## 创建助手

创建助手时至少需要填写 **名称**。常见字段如下。

### 基本信息

- **名称**：群聊中 `@` 时使用的名字，建议短且明确。
- **头像**：用于区分助手。
- **描述**：说明助手擅长什么。
- **分类**：把助手归到某个分类，便于列表管理。

### Agent

Agent 是助手背后的执行工具。当前界面会列出已检测到的 ACP 工具，例如 Claude、Codex。

- 未安装的工具会显示 **未安装**，不能直接选择。
- 如果使用 Claude 或 Codex，可以配置本地工具模型或选择模型供应商。

### 模型供应商

模型供应商来自 **模型** 页面。

- 可以选择兼容当前 Agent 的模型。
- 如果本地 Agent 配置可用，可以选择 **使用本地 Agent 配置**。
- 如果没有本地配置，也没有兼容供应商，助手无法正常调用模型。

### 思考模式

Claude 和 Codex 类型助手支持思考模式：

- **高（默认）**
- **中**
- **低**
- **关**

思考模式越高，复杂任务表现可能更好，但响应时间和 token 消耗也可能增加。

### Codex Fast 模式

Codex 助手支持 **Fast 模式**。开启后速度更快，但 token 消耗速度也会变快。适合短任务或需要快速反馈的场景。

### 本地模型和代理

当使用本地 Agent 配置时：

- Codex 可以选择 Codex 模型，也可以使用本地默认模型。
- Codex 可以填写代理地址，例如 `http://127.0.0.1:7890`。
- Claude 可以选择 Claude 模型，也可以使用本地默认模型。

### 图片生成能力

助手可以启用图片生成能力。启用后必须选择一个已启用的图片模型，否则无法保存。

## 提示词

提示词定义助手的角色、边界和输出风格。建议包含：

- 助手身份和职责。
- 能处理的任务类型。
- 需要遵守的项目规范。
- 输出格式要求。
- 遇到不确定信息时如何处理。

助手详情页的 **配置信息** 中可以编辑系统提示词，并支持 **AI 优化**。

## 助手详情页

### 配置信息

可以查看：

- 系统提示词。
- 助手类型。
- Agent 工具。
- 思考模式。
- 模型供应商。
- 工作目录。
- 创建和更新时间。

非系统助手可以编辑提示词。

### 语音

语音页用于配置朗读和语音相关能力。需要先在模型管理中准备语音模型。

### Skills

Skills 页显示已安装技能，可以安装新 Skill 或卸载已有 Skill。

### 会话记录

会话记录页显示该助手的快速对话历史。点击 **打开** 可以回到对应快速对话群聊。

## 启用和停用

助手详情页可以启用或停用助手。

- 停用后，助手不会作为可用助手参与新选择。
- 已存在群聊中的配置可能仍显示历史数据，但不建议继续依赖停用助手执行任务。

## 使用建议

- 不要把所有职责塞给一个助手，按角色拆分更稳定。
- 需要读写本地项目时，确认助手或群聊工作目录正确。
- 复杂代码任务优先使用 Claude 或 Codex ACP 助手。
- 固定流程建议沉淀成技能后安装到助手。
