# 定时任务

定时任务可以在群聊中按计划自动发送消息，并根据选择触发一个或多个助手。

## 页面入口

进入普通群聊，点击右上角 **更多** → **定时任务**。右侧面板会显示当前群聊的定时任务列表。

快速对话更适合临时任务，不建议作为定时任务承载空间。

## 创建任务

点击 **创建任务**，填写：

- **任务名称**：必填，例如 `每日提醒`。
- **描述**：可选，用于说明任务用途。
- **调度类型**：Cron 表达式、固定间隔、一次性执行。
- **执行内容（发送的消息）**：必填。
- **触发助手**：选择具体助手或所有助手。
- **最大重试次数**：默认 3，范围 0 到 10。
- **立即启用**：创建后是否马上进入调度。

执行内容不要手动写 `@助手名`，系统会根据下方选择自动处理。

## 调度类型

### Cron 表达式

格式是：

```text
分钟 小时 日 月 星期
```

常用预设：

| 预设 | 表达式 |
| --- | --- |
| 每小时 | `0 * * * *` |
| 每天 9:00 | `0 9 * * *` |
| 每天 18:00 | `0 18 * * *` |
| 每周一 9:00 | `0 9 * * 1` |
| 每周五 18:00 | `0 18 * * 5` |
| 每月 1 号 9:00 | `0 9 1 * *` |

### 固定间隔

填写间隔分钟数。例如填写 `60` 表示每 60 分钟执行一次。

### 一次性执行

选择一个具体日期和时间。任务只在该时间执行一次。

## 触发助手

可以选择：

- **所有助手**：排除系统内置助手，对群内普通助手逐条发送消息。
- **具体助手**：多选一个或多个助手。

多选或选择所有助手时，任务执行会逐条发送消息。

## 任务列表

任务卡片会显示：

- 任务名称和禁用状态。
- 调度描述。
- 上次执行时间。
- 下次执行时间。
- 执行内容摘要。
- 上次错误信息。

操作按钮：

- **启用 / 禁用**
- **测试执行**
- **查看历史**
- **编辑**
- **删除**

## 执行历史

点击 **查看历史** 可以看到最近执行记录。

记录包含：

- 状态：待执行、执行中、已完成、失败、已跳过。
- 触发时间。
- 执行内容快照。
- 开始时间、完成时间、耗时。
- 执行记录 ID。
- 错误信息。

## 使用建议

- 把定时任务放在有清晰群规则和工作目录的群聊里。
- 任务内容写成完整指令，不依赖用户临时补充上下文。
- 对会修改文件或运行命令的任务，先用 **测试执行** 验证。
- 如果任务失败，先看执行历史，再看群聊中的执行详情。
