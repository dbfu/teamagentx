# 模型管理

模型管理用于维护 TeamAgentX 调用 AI 服务所需的配置。当前页面支持文本、图片、语音和视频类型，其中文本、图片、语音已有明确配置入口。

## 页面入口

左侧点击 **模型** 进入模型管理页。页面顶部提供：

- **搜索模型名称、API**：按名称或 API 地址过滤。
- **刷新**：重新加载模型列表。
- **导出 JSON**：导出当前模型配置。
- **导入 JSON**：从文件导入模型配置。
- **新增模型**：手动创建配置。
- **AI创建**：当已有默认模型时，可用自然语言描述 API 信息，由 AI 自动填充表单。

## 模型列表

列表支持按类型筛选：

- **全部**
- **文本**
- **图片**
- **视频**
- **语音**

每行会显示模型配置名、模型 ID、API URL、Token 使用情况、启用状态和操作按钮。

常用操作：

- **测试连接**：检查当前配置是否能连通。
- **编辑**：修改名称、API、模型、协议等信息。
- **复制配置**：以当前配置为模板创建副本。
- **设为默认**：把启用中的模型设为默认模型。语音模型中可设置默认 STT。
- **停用 / 激活**：临时让模型不可选或重新可用。
- **删除**：删除配置；关联助手会失去该模型配置。

## 新增文本模型

点击 **新增模型**，选择 **文本**。

必填项：

- **名称**：例如 `Claude Sonnet`、`DeepSeek Chat`。
- **API URL**：例如 `https://api.anthropic.com` 或 OpenAI 兼容服务地址。
- **API Key**：服务密钥。
- **模型**：服务商提供的模型 ID。
- **API 协议**：
  - **Anthropic**：适合 Claude 原生接口，支持 thinking、prompt caching 等 Claude 特性。
  - **OpenAI**：适合 OpenAI 和兼容服务。

可选项：

- **设为默认模型**：让新建助手或需要默认文本模型的功能优先使用它。

## 新增图片模型

选择 **图片** 后，需要配置：

- **图片供应商**：OpenAI、APIMart、OpenRouter、Gemini、Zhipu、Bailian、xAI、Volcengine Ark 或 Custom。
- **调用方式**：
  - **同步返回图片**：请求直接返回图片结果。
  - **返回任务 ID 后轮询**：先提交任务，再查询任务结果。
  - **自动识别**：由系统根据供应商能力处理。
- **API URL**：只填写 base URL，系统会自动追加提交、轮询、取消任务路径。
- **API Key**
- **模型**

注意：

- OpenRouter 的图片模型必须填写支持图片输出的模型 ID，不能使用普通文本模型。
- 百炼 / 万相推荐使用 `wan2.6-t2i` 这类图片模型。
- 智谱推荐填写 `glm-image` 这类图片模型 ID。
- xAI 当前建议使用面向图片生成的新模型。

## 新增语音模型

选择 **语音** 后，需要先选择用途：

- **仅 TTS 朗读**
- **仅 STT 识别**
- **TTS + STT**

语音模型使用 OpenAI 兼容协议：

- 朗读接口会自动追加 `/audio/speech`。
- 识别接口会自动追加 `/audio/transcriptions`。

填写项：

- **API URL**：只填 base URL。
- **API Key**
- **TTS 模型（朗读）**
- **STT 模型（识别）**

如果用途是 TTS + STT，STT 模型留空时会复用 TTS 模型。

## 导入和导出

导出 JSON 适合备份和迁移。导入时系统会校验必填字段：

- 名称
- API URL
- API Key
- 模型

图片模型还需要包含图片供应商和调用方式。导入失败时页面会提示成功和失败数量。

## 使用建议

- 先创建一个稳定的默认文本模型，再创建助手。
- 图片和语音模型只在对应能力需要时配置。
- 修改 API Key 时不要保留遮罩后的密钥文本，必须输入完整密钥。
- 停用模型比删除更适合临时排查问题。
