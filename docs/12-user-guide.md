# 12 · TeamAgentX 使用说明：新手快速上手

> 面向第一次使用 TeamAgentX 的用户。读完本文，你应该能完成初始化、创建助手和群组，并用 `@助手名` 发起一次可追踪的多 Agent 协作。

## 1. 先理解 5 个概念

TeamAgentX 的使用方式接近“把多个 AI 同事拉进一个项目群”。

| 概念 | 通俗理解 | 你需要做什么 |
|------|----------|--------------|
| 模型 | AI 的大脑，如 Claude、OpenAI 兼容模型、DeepSeek 等 | 配置 API Key / API URL / 模型名，或使用本地 Claude/Codex 配置 |
| 技能 | 助手的专项能力包，如代码审查、写文档、处理图片 | 给需要的助手安装技能 |
| 助手 | 一个有角色、有模型、有提示词、有技能的 AI 成员 | 创建或编辑助手，例如“工程师”“质检”“文档员” |
| 群组 | 一个项目工作空间，包含成员、规则、工作目录和消息历史 | 为一个项目建一个群，把相关助手拉进去 |
| 工作目录 | 助手实际读写文件的本地目录 | 创建群时选择项目目录，避免助手改错地方 |

最常用的交互只有一个：在群里输入 `@助手名 任务内容`。

## 2. 推荐上手路径

第一次使用建议按下面顺序完成：

1. 打开应用，完成首次设置。
2. 配置或确认默认 AI 工具 / 模型。
3. 创建至少一个助手。
4. 创建一个群组，选择工作目录并添加助手。
5. 在群里 `@助手名` 下达任务。
6. 通过任务看板、执行记录、上下文面板查看进度和问题。

如果只是临时问一个助手问题，可以跳过建群，直接使用“快速对话”。

## 3. 首次设置

第一次进入系统会看到初始化向导：

1. 点击“开始设置”。
2. 在“工具检测”中选择默认引擎。系统会检测本机 Claude / Codex 等 AI 工具。
3. 在“模型配置”中选择使用本地配置，或手动填写 API 配置。
4. 创建本地账号，填写用户名、密码和头像。
5. 完成后进入主界面。

如果没有检测到本地 AI 工具，可以先安装工具，或稍后在“模型”页面手动添加 API 模型。

## 4. 主界面导航

左侧导航包含主要功能入口：

| 入口 | 用途 |
|------|------|
| 群聊 | 查看项目群、创建群组、聊天协作 |
| 助手 | 创建、编辑、分类管理助手，发起快速对话 |
| 技能 | 查看共享技能、导入外部技能、安装技能到助手 |
| 模型 | 管理文本、图片、语音等模型配置 |
| 设置 | 修改个人信息、主题、提示音、移动端连接和本地工具运行方式 |

## 5. 配置模型

进入“模型”页面后，可以：

- 点击“新增模型”手动添加模型配置。
- 点击“AI创建”，把 API 信息用自然语言粘贴进去，让系统自动解析表单。
- 导入 / 导出 JSON，批量迁移模型配置。
- 测试连接，确认 API Key、API URL 和模型名可用。
- 设置默认模型，供新助手或系统助手优先使用。

文本助手常用配置项：

| 字段 | 说明 |
|------|------|
| 名称 | 自己识别用，例如“DeepSeek Chat” |
| 协议 | Claude 选 Anthropic；OpenAI 兼容服务选 OpenAI |
| API URL | 服务商提供的接口地址 |
| API Key | 服务商密钥 |
| 模型 | 模型 ID，例如 `deepseek-chat`、`gpt-4.1` 等 |

如果你使用本机 Claude Code / Codex，可以在助手配置里选择“不绑定模型供应商，使用本地 Agent 配置”。

## 6. 创建助手

进入“助手”页面，点击“创建助手”。

建议先创建 3 类助手：

| 助手 | 适合做什么 | 提示词重点 |
|------|------------|------------|
| 总控 | 理解需求、拆任务、协调其他助手 | 先澄清目标，再拆解步骤，必要时 @ 其他助手 |
| 工程师 | 写代码、改文件、跑命令 | 严格遵守工作目录，完成后说明改了什么和怎么验证 |
| 质检 | 审查实现、找问题、给验收意见 | 优先找 bug、风险、遗漏测试，不直接改代码 |

创建助手时重点填写：

1. 名称：群聊里会用 `@名称` 触发。
2. 模型供应商：选择一个可用模型，或使用本地 Agent 配置。
3. 描述：方便自己识别助手用途。
4. 提示词：定义助手角色、工作边界和输出格式。
5. 技能：按需安装专项能力。
6. 工作目录：可留空，通常由群组工作目录统一控制。

提示词可以先写简单版本，再用“AI 优化”改写。

## 7. 管理技能

进入“技能”页面可以查看已安装技能。

常见操作：

- “导入外部技能”：扫描外部技能目录并导入。
- “创建符号链接”：适合复用外部技能目录，外部更新会同步。
- “完全复制”：适合固定保存一份技能，不受外部目录影响。
- “安装到助手”：把某个技能分配给一个或多个助手。
- 在群聊中 `@技能管理`：让系统助手协助创建新技能。

使用建议：

- 不要给所有助手安装所有技能。技能越多，助手越容易偏离角色。
- 给工程师装代码相关技能，给质检装 review / verify 类技能，给文档员装写作类技能。

## 8. 创建群组

进入“群聊”，点击“创建群组”。

创建时需要填写：

| 字段 | 建议 |
|------|------|
| 群组名称 | 使用项目或任务名称，例如“官网重构” |
| 群组描述 | 写清这个群的目标 |
| 工作目录 | 选择项目根目录。桌面端可以直接选择文件夹 |
| 选择助手 | 添加总控、工程师、质检等成员 |
| 注入群历史消息 | 默认开启。需要让新助手理解上下文时开启；只想让助手看新任务时关闭 |

群组创建后，群内所有助手共享群工作目录。建议一个项目一个群，不要把多个无关项目放在同一个群里。

## 9. 配置群规则

打开群顶部的设置或“群规则”，可以编辑群规则。群规则会注入到群内所有助手上下文中。

可直接使用下面模板：

```text
- 所有回复使用中文。
- 执行前先说明计划，执行后总结修改内容和验证结果。
- 不确定需求时先提问，不要自行扩大范围。
- 修改代码必须优先遵守当前仓库已有风格。
- 遇到需要用户决策、权限不足、命令失败或测试无法通过时，明确说明阻塞原因。
- 质检只做审查和建议，除非用户明确要求，不直接修改代码。
```

## 10. 选择助手触发模式

群设置里有“助手触发模式”：

| 模式 | 行为 | 适合场景 |
|------|------|----------|
| 协调模式 | 系统内置协调助手接收未 `@` 的消息并派发助手 | 新手推荐，适合多数群组 |
| 自由协作 | 助手消息中的 `@` 会触发其他助手 | 熟悉后使用，适合多助手自动接力 |
| 手动模式 | 助手消息中的 `@` 只显示提及，不触发执行 | 想完全由用户控制调度时使用 |

新手建议使用“协调模式”。如果你发现助手之间互相触发太多，切到“手动模式”。

## 11. 发起一次群聊协作

在群输入框中输入：

```text
@总控 请帮我检查这个项目的启动方式，整理出本地开发步骤，并让工程师验证能否启动。
```

更直接的任务可以这样写：

```text
@工程师 请阅读 README 和 package.json，告诉我如何启动 Web 和后端服务。不要修改文件。
```

如果群设置了默认接收助手，未 `@` 的消息也会自动交给该助手。但为了减少歧义，重要任务建议明确 `@`。

发送后你可以观察：

- 消息气泡里的流式输出。
- 工具调用过程。
- 助手是否处于执行中、排队中或已完成。
- 右侧面板里的任务队列、执行记录和上下文。

## 12. 使用快速对话

快速对话适合 1v1 临时问题，不需要创建正式群。

操作方式：

1. 进入“助手”页面。
2. 找到目标助手。
3. 点击“快速对话”。
4. 可选择工作目录；留空时系统会创建独立会话目录。
5. 进入对话后直接发送消息，不需要 `@`。

适合快速对话的场景：

- 问一个概念。
- 让某个助手临时分析一段文本。
- 测试一个助手提示词是否有效。

不适合快速对话的场景：

- 多助手协作。
- 需要长期保留项目上下文。
- 需要围绕固定项目目录持续工作。

## 13. 查看任务与执行过程

群顶部常用按钮：

| 功能 | 用途 |
|------|------|
| 成员 | 查看群内助手，进入助手详情 |
| 添加助手 | 给当前群添加新助手 |
| 任务看板 | 查看待处理、执行中、已完成、失败/取消、待恢复任务 |
| 停止所有任务 | 有任务失控或不想继续时停止当前群执行 |
| 定时任务 | 创建和管理群级自动任务 |
| 截图聊天记录 | 导出当前聊天记录图片 |
| 清空消息 | 清理当前群消息 |
| 群设置 | 修改名称、工作目录、触发模式、默认助手、群规则 |

在助手详情中常用：

- “查看上下文”：看助手实际收到的上下文。
- “执行历史”：查看历史执行记录。
- “任务队列”：查看该助手当前排队和执行情况。
- “清空上下文”：重置该助手在当前群的记忆。

当助手答非所问、记住了错误信息或上下文太乱时，优先使用“清空上下文”。

## 14. 创建定时任务

群级定时任务可以定时向群里发送消息并触发助手。

操作方式：

1. 打开群顶部“定时任务”。
2. 点击“创建任务”。
3. 填写任务名称和描述。
4. 选择调度类型：Cron 表达式、固定间隔或一次性执行。
5. 填写执行内容。
6. 选择要触发的助手。不要在执行内容里手写 `@助手名`，系统会根据选择自动处理。
7. 设置最大重试次数和是否立即启用。

例子：

| 场景 | 配置 |
|------|------|
| 每天早上同步项目状态 | 每天 9 点，触发总控，总结昨日进展和今日待办 |
| 每周写变更总结 | 每周五 18 点，触发文档员，生成 changelog 草稿 |
| 定期巡检服务 | 固定间隔，触发监控助手，检查服务状态 |

## 15. 工作目录使用建议

工作目录决定助手能在哪个项目里运行命令、读写文件。

推荐做法：

- 一个项目建一个群，并把工作目录设为项目根目录。
- 不要把工作目录设为用户主目录或过大的父目录。
- 多个助手需要协作同一项目时，放在同一个群里。
- 临时实验使用快速对话的独立目录，避免污染正式项目。
- 桌面端可以复制、打开工作目录；Web 版受浏览器限制，目录能力会少一些。

## 16. 移动端连接

在“设置”中找到“移动端连接”：

1. 确认电脑和手机在同一局域网。
2. 选择可用的局域网地址。
3. 生成二维码。
4. 用手机 TeamAgentX App 或扫码入口扫描。
5. 手机端会自动登录并连接对应服务。

如果扫码后连接失败，优先检查：

- 手机和电脑是否在同一网络。
- 设置中选择的局域网 IP 是否正确。
- 防火墙是否阻止访问。
- 桌面端服务是否仍在运行。

## 17. 常见问题

### 助手没有响应

检查：

- 是否明确 `@助手名`。
- 群里是否已经添加该助手。
- 模型配置是否启用且连接测试通过。
- 当前是否有任务排队。
- Socket / 服务端连接是否正常。

### 助手用错项目目录

检查群设置里的“工作目录”。如果目录为空，系统会使用默认群目录。正式项目建议明确选择项目根目录。

### 助手互相触发太多

把群设置里的“助手触发模式”改为“手动模式”，然后由用户显式 `@` 每个助手。

### 助手上下文混乱

进入助手详情，先“查看上下文”确认它看到了什么；必要时点击“清空上下文”。如果是新加入的助手，按需关闭“注入群历史消息”。

### 模型不可用

到“模型”页面检查：

- API URL 是否正确。
- API Key 是否有效。
- 模型 ID 是否拼写正确。
- 协议是否选对，Claude 用 Anthropic，OpenAI 兼容服务用 OpenAI。
- 是否已启用该模型配置。

### 桌面端打不开目录

目录打开能力只在 Electron 桌面端完整支持。Web 版受浏览器限制，可能只能复制路径。

## 18. 新手推荐配置

最小可用组合：

| 项 | 推荐 |
|----|------|
| 模型 | 配一个稳定的文本模型，设为默认 |
| 助手 | 总控 + 工程师 + 质检 |
| 群组 | 一个项目一个群 |
| 触发模式 | 协调模式 |
| 群规则 | 使用第 9 节模板 |
| 工作目录 | 项目根目录 |

第一次验证可以发送：

```text
@总控 请熟悉当前项目，说明它是什么、如何启动、主要目录有什么。先不要修改任何文件。
```

确认助手能正确读取项目、回答启动方式后，再让它执行更具体的开发任务。
