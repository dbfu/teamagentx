# 飞书集成

## 适用模型

飞书接入已切到“平台下多个机器人实例”的模型：

- 一套 `appId + appSecret` 表示一个飞书机器人实例
- 一个飞书机器人实例最多绑定一个 TeamAgentX 群聊
- 一个 TeamAgentX 群聊可以同时连接多个飞书或其他平台机器人

## 需要准备

- 飞书应用 `App ID`
- 飞书应用 `App Secret`
- 可选：默认飞书会话 ID（`chat_id` / `open_chat_id`），用于在飞书侧尚未主动发消息前从 TeamAgentX 主动推送

快捷入口：

- [飞书快捷创建机器人入口](https://open.feishu.cn/page/openclaw?form=multiAgent)

## 推荐接入步骤

1. 在飞书开放平台创建应用
2. 打开机器人能力和事件接收能力
3. 配置应用权限，确保机器人能接收和发送消息
4. 进入 TeamAgentX 集成页面，选择飞书
5. 新建机器人实例，填写 `App ID` 和 `App Secret`
6. 如果希望 TeamAgentX 在飞书侧首次发消息前就能主动推送，填写默认飞书会话 ID
7. 直接选择要绑定的 TeamAgentX 群聊并保存

## 运行行为

- 飞书机器人收到的会话消息会进入绑定群聊
- 绑定群里的消息会优先同步回该飞书机器人最近活跃的会话；如果还没有来源会话，则使用配置的默认飞书会话 ID
- 群里助手开始执行时，系统会在飞书用户原消息上添加 `Typing` 表情反应；回复完成后会自动移除

## 说明

- 当前实现以机器人实例为中心，不再维护“飞书群聊映射”概念
- 飞书输入状态使用消息 Reaction 模拟，效果与 `@larksuite/openclaw-lark` 的工作中标识一致；如果飞书应用缺少表情回复权限，会静默降级为不显示该标识
