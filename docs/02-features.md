## 1. 模型供应商管理 (LLM Provider)

### 1.1 基础 CRUD

| 能力 | 状态 | 说明 |
|------|------|------|
| 添加 / 编辑 / 删除供应商 | ✅ | `POST/PUT/DELETE /llm-providers` |
| 启用 / 停用 | ✅ | `is_active` 字段，`PATCH /llm-providers/:id/status` |
| 设为默认 | ✅ | `is_default` 字段，`PATCH /llm-providers/:id/default` |
| 测试连通性 | ✅ | `POST /llm-providers/:id/test` |

### 1.2 协议与厂商支持

| 协议 | 状态 | 说明 |
|------|------|------|
| Anthropic 协议 | ✅ | 原生支持 Claude 系列 |
| OpenAI 兼容协议 | ✅ | 覆盖 OpenAI、DeepSeek、Kimi、阿里云百炼等 |
| 自定义 endpoint | ✅ | `api_url` + `api_key` 自由配置 |
| 本地 Agent | ✅ | "使用本地 Agent 配置" —— 复用本机 Claude Code / 同类 CLI 的 key |
| ACP 工具协议 | 🟡 | 部分支持，"该 ACP 工具暂不支持自定义 LLM 供应商" |

### 1.3 智能配置（亮点）

- **粘贴文本一键解析配置**（`POST /llm-providers/parse-config`）
  - UI 提示："输入您的 API 配置信息描述，AI 会自动解析并填充表单。例如：我的 Claude API key 是 sk-ant-xxx"
  - 适配场景：用户从文档/聊天记录复制 API 信息，平台自动拆出 name/url/key/model

### 1.4 失效保护

- "已清除不兼容的 LLM 供应商" —— 模型协议不匹配时自动清理引用
- "已失效的默认助手" —— 默认助手依赖的模型失效后有提示
- "暂无兼容供应商，请先配置模型或登录本地 Agent" —— 引导式空状态

## 2. 助手管理 (Agent)

### 2.1 基础 CRUD

| 能力 | 状态 | 说明 |
|------|------|------|
| 创建 / 编辑 / 删除助手 | ✅ | `POST/PUT/DELETE /agents` |
| 助手列表 | ✅ | `GET /agents`、`/agents/active`、`/agents/grouped` |
| 助手排序 | ✅ | `PUT /agents/sort-order` |
| 激活/停用助手 | ✅ | `PATCH /agents/:id/status` |
| 头像 + 颜色 | ✅ | 自定义头像、avatarColor |

### 2.2 助手分级（重要概念）

| `agent_level` | 含义 | 行为 |
|---------------|------|------|
| `system` | 系统预置助手（如"技能管理"、默认总控/工程师/质检模板） | 不允许修改、不允许其他助手拖入、自动出现在所有群成员选择器 |
| `normal` | 用户自建助手 | 完整可编辑 |

UI 反馈：
- "系统助手不允许修改"
- "系统助手不允许其他助手拖入"
- "系统分类不允许添加助手"

### 2.3 助手分类（Categories）

- 用户可创建自定义分类，将助手归类
- 系统分类（system folder）只读
- "创建分类" / "修改分类名称" / "删除分类"
- "请输入分类名称" / "请输入分类描述（可选）"
- "选择分类（可选）"

### 2.4 助手配置三件套

| 配置 | 说明 |
|------|------|
| **模型** | 引用 LLM Provider id（可选默认）|
| **系统提示词** | "请输入助手的提示词，用于定义助手的行为和角色" |
| **技能（Skills）** | 助手可装载多个 skill |

### 2.5 提示词优化（亮点）

- `POST /agents/optimize-prompt` —— 同步优化
- `POST /agents/optimize-prompt-stream` —— **流式优化**，实时看 AI 改写过程
- UI："编辑提示词"、"提示词已优化"

### 2.6 快速对话（Quick Chat）

- `POST /agents/quick-chat` —— 跳过建群直接 1v1 速聊
- `GET /agents/:id/quick-chat-rooms?userId=...` —— 列出该助手的所有快速对话
- 快速对话本质是一个"特殊群聊"：`isQuickChat: true`，"这是一个快速对话群聊，消息会直接发送给助手，无需 @ 提及"
- 快速对话支持自定义 workDir："留空时，每次对话将创建独立的会话目录"

### 2.7 助手默认目录

- 助手级 `workDir` 默认值，被群引用时可覆盖
- "助手默认目录"

## 3. 技能管理 (Skill)

### 3.1 三种安装方式

| 方式 | 说明 | 端点 |
|------|------|------|
| **完全复制** | 文件落本地，独立管理，不受外部更新影响 | `/skills/install` |
| **符号链接（symlink）** | 链接指向外部目录，外部更新自动同步，节省磁盘 | `/skills/symlink` |
| **导入外部** | 一次性扫描外部目录批量导入 | `/skills/import-external`、`/skills/external` |

UI 提示：
- "完全复制技能文件到本地，独立管理，不受外部更新影响。"
- "创建符号链接指向外部技能目录，外部技能更新后自动同步，节省磁盘空间。"

### 3.2 技能市场（雏形）

| 能力 | 状态 | 端点 |
|------|------|------|
| 发现 / 浏览 | ✅ | `GET /skills/discover` |
| 搜索 | ✅ | `GET /skills/search` |
| 批量安装到助手 | ✅ | `POST /agents/:id/skills/install-selected` |
| 共享技能 | ✅ | `/skills/shared` |
| 兼容 Claude Code skill 格式 | ✅ | symlink 时直接挂 `~/.claude/skills/` |

### 3.3 通过聊天创建 Skill（亮点）

- 系统预置助手"**技能管理**"——通过群聊对话生成 skill
- UI 提示："在群聊中 @技能管理 创建技能"、"在群聊中 @技能管理 创建新技能"
- "如：帮我找一个写代码的技能、分析数据的技能、处理图片的技能..."
- 端点：`POST /skills/create`

### 3.4 助手 ↔ 技能关系

- `GET /agents/:id/skills` —— 列出助手装的 skill
- `POST /agents/:id/skills/discover` —— 给该助手推荐相关 skill
- `POST /agents/:id/skills/install` —— 装单个
- `POST /agents/:id/skills/install-selected` —— 批量装
- 一个 skill 可被多个助手共享（通过 `/skills/shared`）

## 4. 群聊管理 (ChatRoom)

### 4.1 基础 CRUD

| 能力 | 状态 | 说明 |
|------|------|------|
| 创建 / 编辑 / 删除群 | ✅ | `POST/PUT/DELETE /chatrooms` |
| 置顶 / 取消置顶 | ✅ | `PATCH /chatrooms/:id/pin` `unpin` |
| 标已读 | ✅ | `PATCH /chatrooms/:id/mark-read` |
| 全局未读数 | ✅ | `GET /chatrooms/unread-counts` |
| 群描述 | ✅ | "请输入群组描述" |

### 4.2 群成员管理

| 角色 | 含义 | 行为 |
|------|------|------|
| `OWNER` | 群主，必为人类用户 | 群主发言可触发默认助手；不可被移除 |
| `MEMBER` | 普通成员，可以是助手或人类 | 可被移除 |

成员视图分类显示：**Owners / System Agents / Normal Agents** 三组。

### 4.3 助手个性化（群级覆盖）

每个 `chatRoomAgent` 关系上可独立配置：

| 字段 | 含义 |
|------|------|
| `injectGroupHistory` | 加入时是否注入群历史消息（**上下文管理关键**）|
| 助手在群内的设置 | `/chatrooms/:id/agents/:id/settings` |
| 清空助手在该群的上下文 | `POST /chatrooms/:id/agents/:id/clear-context` |
| 查看助手在该群的上下文 | `GET /chatrooms/:id/agents/:id/context` |
| 查看助手在该群的执行记录 | `GET /chatrooms/:id/agents/:id/executions` |
| 该助手在该群分到的任务 | `GET /chatrooms/:id/agents/:id/tasks` |

UI 提示：
- "开启后助手可以查看群聊历史消息" / "已开启注入群历史" / "已关闭注入群历史"
- "开启后获取群聊上下文"
- "已清空对话上下文"

### 4.4 触发模式（重要设计）

| 模式 | 行为 | UI 提示 |
|------|------|---------|
| **自动模式（推荐）** | 助手消息中的 @ 会触发其他助手执行任务 | "自动模式：助手消息中的 @ 会触发其他助手执行任务。" |
| **手动模式** | 助手消息中的 @ 不会触发其他助手，仅作提及 | "手动模式：助手消息中的 @ 不会触发其他助手，仅作提及。" |

→ 自动模式适合"协作流水"，手动模式适合"我作为用户调度，助手不串台"。

### 4.5 默认接收助手

- "群主发送未 @ 助手的消息时，会自动触发该助手。"
- 解决问题 C（@ 触发歧义）的现有方案——用户随手一句没 @ 的消息，由默认助手兜底

### 4.6 群规则

- "群规则会注入到群内所有助手的上下文中，指导助手的行为。"
- 单条 rule 添加 / 编辑（"暂无规则，点击添加"）
- 保存时整体下发（"群规则已保存"）

### 4.7 快速对话子会话

- `POST /chatrooms/:id/quick-chat-session` —— 在群下挂一个快速会话
- "通过快速对话功能创建的会话记录"
- "创建新的临时会话"

## 与原始描述的对照

用户原始描述：
> 模型管理可以添加各种模型、技能管理安装 skill、助手管理配置助手（助手可以配置模型，提示词和安装 skill 技能）、消息管理可以拉多个助手拉起群聊（群中可以配置群规则、群规则规范每一个助手、群通过 @ 触发助手工作）

本文档发现**实际产品比描述丰富得多**——以下是用户描述里**未提到但已实现**的能力：
