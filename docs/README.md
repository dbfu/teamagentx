## 这套文档讲什么

把 **TeamAgentX**（一个让单个开发者就能调度多 AI 助手的群聊式协作平台）从"是什么"、"怎么干活"、"踩了哪些坑"、"跟同行比怎么样"、"下一步往哪走"五个维度讲透。

兼顾三种读者：

- **设计文档读者**（开发团队）：架构、schema、状态机、接口
- **对外讲解读者**（投资人、潜在用户）：定位、隐喻、差异化
- **深度思考读者**（产品负责人）：问题诊断、解决路径、决策记录

## 核心结论速览

### 1. 产品定位
**让一个开发者拥有"能调度的 AI 团队"**——这是与 Cursor（一个超级员工）、Devin（黑盒自主代理）、CrewAI（写代码定义流程）的根本区别。

### 2. 三大护城河
1. **群聊隐喻**——零学习成本（人人会用群）
2. **任务卡 + 群即项目**——让乱聊变协作，文件/任务/记忆三层隔离
3. **可观测的多 Agent 协作**——流式 thinking + 执行记录 + 上下文检视

### 3. v0.1.0 已落地（远超用户最初描述）
11 大功能域、约 60+ API 端点、若干独有亮点：
- ⭐ **群级 Cron 定时任务**（独有）
- ⭐ **粘贴文本一键解析模型配置**（UX 亮点）
- ⭐ **流式提示词优化**
- ⭐ **通过聊天创建 Skill**（"@技能管理"系统助手）
- ⭐ **Skill symlink 模式**（兼容 Claude Code）
- ⭐ **三层 workDir 策略**
- ⭐ **每个助手在每个群独立的上下文 / 历史注入开关**
- ⭐ **本地 Agent 复用**（接 Claude Code key）
- ⭐ **移动端扫码连接**
- ⭐ **Bridge 外部平台**（Telegram / 飞书 / 钉钉 / 企业微信 接入同一群聊）
- ⭐ **思考模式（thinkingMode）**：off/low/medium/high 控制 Claude 推理预算

详见 [02-features.md](02-features.md)。

### 4. 13 个重大问题点
按 4 个主题归类，按优先级排序：

| 优先级 | 数量 | 问题 |
|-------|------|------|
| 🔴 P0 | 5 | A1 防循环、**A2 扇出风暴**、D 文件冲突、E 信息同步、G 状态漂移 |
| 🟡 P1 | 7 | B 上下文爆炸、C @歧义、F 角色越界、H 死锁、I 中断恢复、J 模型错配、K 人类介入、L 群规则 |
| 🟢 P2 | 1 | M 评估盲区 |

完整方案见 [04-problems-and-solutions.md](04-problems-and-solutions.md)。

### 5. 阶段一关键 Milestone：硬化任务卡协作

5 项 P0 任务联动做（**联动 1**），一鱼多吃同时解决 4 个问题：

```
T1 任务卡 schema 定稿  →  所有改造的地基
T2 扇出风暴防御       →  解决 A2（用户重点提）+ A1
T3 任务卡 ↔ git 分支  ┐
T4 文件变更事件总线   ├ 联动 1：解决 D / E / G / I 四个问题
T5 客观验收钩子       ┘
```

详见 [06-roadmap.md](06-roadmap.md)。

### 6. 不该学的避坑
- 完全自主"超级员工"（Devin 路线）—— 押多 Agent 协作的可观测性
- 拖拉拽 DAG（Dify/Coze 路线）—— 押对话式动态调度
- 代码定义流程（CrewAI/LangGraph 路线）—— 押 GUI 配置
- 锁死在编辑器（Cursor/Copilot 路线）—— 走出 IDE 做"虚拟办公室"

详见 [05-competitors.md](05-competitors.md) 第 5 节。
