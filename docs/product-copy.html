AI 数字员工协作空间 · 群聊式多 Agent 工作台

让 AI 不再单兵作战，而是像团队一样协作交付。

TeamAgentX 把“群聊、@ 触发、任务队列、工作目录、执行记录、技能和定时任务”组合成一个可操作的 AI 团队。用户只要像在工作群里一样 @ 对的人，就能调度总控、工程师、质检、资料员和文档员一起完成复杂任务。

为什么现在需要 TeamAgentX

单 Agent 很强，但复杂工作天然需要分工。

需求理解、资料查找、代码实现、质量审查、文档沉淀不是同一种能力。把所有事情塞给一个聊天窗口，会让上下文越来越重、角色越来越乱、过程越来越不可追踪。

01

上下文爆炸

同一个 Agent 反复读取历史消息、文件细节、错误记录和补充说明，token 被大量消耗在无效上下文里，越到后面越难保持稳定判断。

02

角色边界混乱

让一个 Agent 同时做产品、工程、测试、文档，很容易该质疑时直接执行，该验收时继续发散，团队经验也无法稳定复用。

03

工作流门槛高

传统 Agent 工作流需要配置节点、变量和工具协议。普通用户真正熟悉的协作方式，是在群里说清楚目标，然后 @ 对的人。

TeamAgentX 的产品判断：不要让用户学习复杂编排。把 AI 员工放进项目群，用群聊完成调度，用任务队列承接执行，用上下文隔离控制成本，用执行记录建立信任。

四个产品爆点

不是概念包装，每个爆点都有真实功能支撑。

TeamAgentX 的差异化不是“让几个机器人聊天”，而是把多 Agent 协作做成一套看得见、可控制、能沉淀的产品系统。

爆点 01

Agent 自由协作，像工作群一样自然

用户通过 @ 触发指定助手。自动模式下，助手回复里的 @ 可以继续触发其他助手，形成总控派单、工程执行、质检验收的自然接力。

  • @ 助手触发任务
  • 协调 / 自由协作 / 手动模式
  • 默认接收助手兜底
  • 群规则统一注入
爆点 02

分工减少上下文爆炸，省 token

不再把全部历史塞给一个模型。每个助手有自己的角色视角、记忆切片和群内上下文，新助手是否看历史也可以单独控制。

  • 每个群独立工作目录
  • 助手上下文独立保存
  • 群历史注入开关
  • 查看 / 清空上下文
爆点 03

数字员工，蒸馏同事经验

助手不是简单 prompt，而是模型、提示词、技能、角色边界、工作目录和执行历史的组合，可以把资深同事的做事方式沉淀成可复用 Agent。

  • 助手分类、排序、启停
  • 独立模型和提示词
  • AI 优化提示词
  • 安装、共享、创建 skill
爆点 04

小白友好，复杂能力藏在熟悉交互里

用户不需要理解多 Agent 框架，只需要建群、拉助手、选目录、发消息。执行中、排队中、工具调用和失败记录都在界面里可见。

  • 飞书/Lark 风格 UI
  • AI 解析模型配置
  • 任务看板和执行记录
  • 桌面端 + 移动端连接
产品形态

一个群，就是一个 AI 项目办公室。

在 TeamAgentX 里，群不是聊天容器，而是项目容器。它绑定成员、群规则、工作目录、消息历史、任务队列、执行记录和定时任务。助手在群里协作，文件在目录里落地，过程在任务和记录里追踪。

这让产品既有聊天的低门槛，也有工作台的可控性：用户可以随时插话、打断、改派、查看上下文，避免黑盒 Agent 失控。

真实功能模块

核心能力不仅讲得清楚，演示时也能点得出来。

下面这些能力都来自当前系统：模型、助手、技能、群聊、消息、任务、定时任务、工作目录和多端连接共同构成完整闭环。

@

群聊式调度

通过 @助手名 触发工作，支持默认助手、协调模式、自由协作和手动模式。

  • 多助手在同一群协作
  • 群规则注入所有助手
  • 任务队列避免抢答
AI

数字员工配置

助手可绑定模型、提示词、技能、头像、分类、默认目录，并可用 AI 优化提示词。

  • 系统助手和自建助手
  • 助手分类与排序
  • 快速对话 1v1 入口
SK

技能沉淀

支持导入外部 skill、符号链接、完全复制、共享技能和批量安装到助手。

  • 兼容 Claude Code skill 生态
  • @技能管理 创建新技能
  • 按角色装载专项能力
TQ

执行可观测

实时展示流式输出、思考内容、工具调用、执行状态和完成事件。

  • 查看消息执行详情
  • 查看助手任务队列
  • 查看和清空上下文
CR

群级定时任务

用 cron、固定间隔或一次性任务，让数字员工按时间自动进入群里工作。

  • 立即测试运行
  • 启用 / 停用
  • 历史执行记录
WD

工作目录隔离

群级共享目录、助手默认目录、快速对话独立目录，避免临时任务污染正式项目。

  • 桌面端打开本地目录
  • 复制路径和文件预览
  • 移动端扫码连接
一条可演示的协作链路

从一句需求，到一组数字员工接力交付。

这条链路把“多 Agent 协作”拆成用户能直接理解和观察的步骤：建群、设规则、派单、执行、追踪、验收。

01

建群

为一个项目创建群组，选择工作目录，把总控、工程师、质检、文档员加入群。

02

设规则

配置群规则，统一要求助手先拆解、再执行、最后说明验证结果。

03

@ 总控

用户一句话描述目标，总控理解需求，拆分任务，并 @ 对应助手。

04

队列执行

助手进入任务队列，按房间和助手上下文顺序处理，避免抢答和串台。

05

过程可见

用户看到流式输出、工具调用、任务状态、执行记录和助手实际上下文。

06

验收沉淀

质检审查，文档员总结，相关方法沉淀为提示词、技能和群规则。

使用场景

同一套能力，覆盖个人、团队和企业知识沉淀。

TeamAgentX 可以先从个人开发者的 AI 小队切入，再扩展到团队流程标准化和企业数字员工资产。

个人开发者

一人维护多个项目时，把每个项目变成一个群，让总控拆任务、工程师改代码、质检做审查。

  • 项目级工作目录
  • 快速对话处理临时问题
  • 执行记录方便复盘

研发团队

把研发协作流程做成群规则和助手编队，新项目直接复用总控、前端、后端、测试、文档角色。

  • 不同助手绑定不同模型
  • 按角色安装技能
  • 任务看板追踪交付

运营内容团队

竞品扫描、日报周报、资料整理、发布说明都可以交给群级定时任务和不同助手接力完成。

  • cron 周期执行
  • 聊天记录截图导出
  • 资料员和文档员分工

企业知识沉淀

把资深同事的流程、标准、口吻和判断方式沉淀为助手提示词和 skill，新人可以直接 @ 数字员工。

  • 助手分类管理岗位
  • 技能共享和复用
  • 群历史注入控制边界

把 AI 从“聊天工具”升级为“数字员工团队”。

TeamAgentX 围绕协作、上下文控制和数字员工沉淀构建产品闭环。它不是只解决“问答效率”，而是在重构个人和团队使用 AI 完成复杂工作的方式。
