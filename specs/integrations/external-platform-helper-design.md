# 外部平台接入助手设计

## 目标

把“外部平台接入”做成一个群里可直接使用的系统助手，而不是只靠后台页面操作。

新的目标不是创建“外部平台群映射”，而是：

- 在当前 TeamAgentX 群里拿到平台凭证
- 创建对应平台的机器人实例
- 直接把这个机器人实例绑定到当前群聊

## 助手职责

系统助手名称：`外部平台接入`

职责：

1. 告诉用户某个平台需要准备哪些凭证
2. 告诉用户这些凭证去哪里拿
3. 接收用户给出的平台凭证
4. 创建机器人实例
5. 把机器人实例绑定到当前群聊
6. 返回绑定结果和后续需要做的少量动作

## 主流程

### 场景 1：用户在群里接 Telegram

1. 用户说：`@外部平台接入 把这个群接到 Telegram`
2. 助手返回需要的凭证：Bot Token
3. 用户把 Token 发给助手
4. 助手创建一个 Telegram `BridgeBot`
5. 助手把该 `BridgeBot` 绑定到当前群聊
6. 助手返回“已绑定成功”，必要时附带 webhook 或下一步说明

### 场景 2：用户在群里查看当前连接

1. 用户说：`@外部平台接入 看看这个群接了哪些机器人`
2. 助手查询当前群聊下所有 `BridgeBot`
3. 按列表返回：名称、平台、状态

## 不再采用的旧流程

以下旧思路现在不再适用：

- 让用户先提供外部群 ID
- 创建“外部群 -> 内部群”的映射关系
- 用平台级唯一配置去覆盖整个接入
- 把默认回复助手挂在机器人实例上

## 与 UI 的关系

助手完成的动作应当和集成页面完全一致：

- 创建机器人实例
- 绑定当前群聊
- 在集成页面可见
- 在群设置页面可见

也就是说，群里通过助手接入，和后台通过页面接入，最终都写入同一套 `BridgeBot` 数据。
