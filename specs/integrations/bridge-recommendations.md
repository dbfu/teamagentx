# TeamAgentX 外部平台桥接建议

## 当前产品原则

外部平台桥接现在应该坚持这几个原则：

1. TeamAgentX 群聊是主协作空间
2. 外部平台机器人是通信入口和传播出口
3. 不再抽象“外部平台群聊映射”层
4. 一个群聊可以同时连接多个平台机器人
5. 机器人不决定默认助手，群聊自己决定

## 推荐用户心智

不要再把这套能力对外描述成：

- “配置某个平台”
- “绑定一个外部群”

建议统一描述成：

- “把 AI 团队接进这个群”
- “给这个群连接一个 Telegram / 飞书 / 钉钉机器人”
- “这个群已经连了哪些外部机器人”

## 推荐交互

### 频道页面

- 平台下先看到“机器人实例列表”
- 每个实例只展示最核心的两行信息
- 创建机器人时支持直接选群聊
- 同一群聊允许重复出现多个不同实例

### 群设置页面

- 展示“已连接机器人列表”
- 每行：机器人名 / 平台 / 状态 / 解绑
- 不再用高卡片展示单一绑定结果

### 群里系统助手

- 用户只需要给出平台凭证
- 助手自动创建实例并绑定当前群聊
- 用户不需要理解数据库模型

## 推荐运行策略

### 入站

- 外部消息进入哪个机器人实例，就落到它绑定的群聊

### 出站

- 群消息同步到该群下所有“已有活跃来源会话”的机器人
- 助手执行前先同步输入状态
- 助手完成后同步最终结果

## 当前不建议做的事

- 不建议恢复 `ExternalChannel` 这类旧映射模型
- 不建议要求用户必须先执行绑定命令
- 不建议把机器人绑定页做得很高、很重
- 不建议让机器人实例承担助手调度职责
